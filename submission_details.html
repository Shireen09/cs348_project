Project Submission Details

a) SQL Injection Protection

  • The app uses Django ORM queries (filter, get_object_or_404, and model save()), which execute parameterized SQL rather than string-concatenated SQL.
  • Forms include server-side validation and sanitization (for example, clean_code() and clean_name() in CountryForm).
  • CSRF protection is enabled on forms via {% csrf_token %}.

b) Indexes and Query/Report Support

  • country_cont_name_idx on Country(continent, name) supports:
    • Report filtering by continent and ordering by country name in /report/.
    • Main country listing ordered by name in /.
  • lang_country_idx on CountryLanguage(language, country) supports:
    • Report filtering by selected language in /report/.

c) Transactions and Isolation Level

  • Create, edit, and delete operations run inside transaction.atomic() blocks.
  • For multi-user scenarios, edit/delete operations use select_for_update() to lock rows where supported.
  • The app attempts REPEATABLE READ isolation on PostgreSQL/MySQL transactions. SQLite remains fully functional and safely falls back to its default behavior.
